    Cookbooks

    This section showcases practical examples that highlight the core value propositions of the AI Computing Broker (ACB). Cookbook 1 demonstrates how ACB enhances GPU utilization during OpenFold inference. Cookbook 2 shows how ACB manages memory oversubscription by deploying and orchestrating multiple LLMs on a single vLLM server.

    Cookbook 1 Improving GPU Efficiency: In Silico Alanine Scan with OpenFold

    This demo shows how to perform an in silico alanine scan using a modified version of OpenFold. You can screen how single-point mutations affect protein structure without experimental wet lab steps.

    Biological Motivation

    Alanine scanning mutagenesis is a common technique used in molecular biology and drug discovery to evaluate the functional importance of individual residues in a protein. By systematically replacing residues with alanine. This helps for example to identify critical residues for binding or stability.

    We demonstrate how to reclaim idle GPU time during OpenFold's inference by using ACB's agarun auto client. By parallelizing mutant evaluations, we significantly improve GPU utilization and inference throughput compared to the serial baseline.

    How to Run

    1. Generate Data

    create own environment the openfold one is very fragile: 

    conda create -n data_env -c conda-forge datasets
conda activate data_env

    python make_data.py

    This creates a small uniref90_small.fasta file in data/.

    CIF file are already included in data/dummy_templates/.

    2. Install OpenFold

    This should be tagged prob otherwise the api changes the our modifed inference script breaks... 

    git clone https://github.com/aqlaboratory/openfold.git

    cd openfold/
    Once you're in the OpenFold project root directory, continue with the setup.

    1. Make env: 
      conda env create -n openfold_env -f environment.yml

      conda activate openfold_env
    2. Install Openfold's CUDA Extensions: 
      python setup.py install
    3. Download model weights 
      bash scripts/download_alphafold_params.sh openfold/resources
    4. Install additional packages for this demo 
      conda install -c conda-forge openblas notebook matplotlib && pip install py3Dmol
    5. Install ACB in your environment (see Quick Install Guide) Make sure you're on the compatible branch, and modify run_pretrained_openfold.py as per this repo's demo.

    3. Set Up Demo

    Due to OpenFold's use of relative paths, we need to place a modified inference script directly inside the openfold repo:

    cp mod_run_pretrained_openfold.py openfold/
    This files introduces a custom lightweight Jackhmmer implementation to avoid downloading hundreds of gigabytes of data. To better simulate realistic MSA preprocessing time, we introduce synthetic idle time before inference. This is exposed as a tunable idle_time parameter to mimic real-world CPU-bound bottlenecks and show how ACB can help reclaim idle GPU time during such workflows.

    Key changes mod_run_pretrained_openfold.py: - Replaced alignment logic at line 69 with run_jackhmmer(...). - Used custom_alignment_dir = run_jackhmmer(...) at line 363 for custom alignment input. - Added support for: - --jackhmmer_db_dir (custom small database) - --idle_time (simulate MSA latency) - Introduced small helper functions for file handling and alignment.

    Run the minimal demo with a small sequence to verify ACB, serial and parallel run of openfold work as expected: 

    python minimal_demo.py

    Cookbook 2 Managing Memory Oversubscription: Hosting several LLMs on a single vLLM server with ACB

    This guide demonstrates how to deploy and run the AI Computing Broker (ACB) using the vLLM framework to host several LLMs on a single vLLM server.

    Installation Steps

    1. Create a virtual environment: 

      python -m venv .venv
source .venv/bin/activate

    2. Install AI Computing Broker: see Quick Install section in the manual.

    3. Install demo dependencies: 

      pip install llama-index==0.11.2 llama-index-embeddings-ollama llama-index-llms-openai-like

    4. Setting Configuration: Create an .env file under the docker folder: 

      # Proxy settings
http_proxy=
https_proxy=
HTTP_PROXY=
HTTPS_PROXY=
no_proxy=localhost,127.0.0.1
NO_PROXY=localhost,127.0.0.1

# ACB installation directory
ACB_DIR=/path/to/your/acb/installation

# User permissions (run these commands to get values)
UID=XXXX  # Get from: id -u
GID=XXXX  # Get from: id -g

# Hugging Face cache directory
HF_HOME=~/.cache/huggingface

# GPU scheduler type
SCHEDULER=gpu-affinity

    Create cache file for HuggingFace with permission: 

    mkdir -p ~/.cache/huggingface
chmod 755 ~/.cache/huggingface

    Run vLLM Server with ACB

    All required Docker containers are provided. For detailed instructions, refer to the Docker section in the user manual.

    This demo demonstrates usage with Phi-4, TinySwallow-1.5B-Instruct, and TinyLlama-1.1B-Chat-v1.0, integrated with RAG-based data sources.

    cd docker
docker compose up -d

    Simple Run

    python main.py
