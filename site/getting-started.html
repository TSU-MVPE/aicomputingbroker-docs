    AI Computing Broker (ACB) ‚Äî Quickstart Guide

    This guide walks you through the initial setup of the AI Computing Broker (ACB) and helps you run your first job in under 5 minutes.

    Looking for more details?

    For a conceptual overview, refer to the Fujitsu Research Portal. For comprehensive documentation, see the User Manual.

    Step 1: Installation

    Install ACB Package

    pip install ai-computing-broker

    Set Up License

    Choose one of the following methods:

    === "Cloud License (Recommended)"

    Configure the environment variable with your API key:
```bash
export ACB_KEY="YOUR_ACB_KEY_HERE"
```

    === "On-Premises License"

    Place the `license.lic` file in the directory where you'll run `gpu_assigner`.

    Requesting a License

    Both license types are provided by Fujitsu. Request access at the Fujitsu Research Portal.

    Step 2: Launch GPU Assigner

    The GPU Assigner service manages GPU allocation across your jobs.

    Start the Service

    gpu_assigner start

    Expected Output: 

    INFO: Successfully started GPU Assigner on 127.0.0.1:11234

    Verify Status

    gpu_assigner status

    Expected Output: 

    INFO: GPU assigner is running on 127.0.0.1:11234 (pid=13412)

    License Required

    Ensure the license file is present in the working directory or ACB_KEY is set before starting the GPU Assigner.

    Step 3: Run Your First Job

    Install PyTorch (if needed)

    pip install torch

    Create a Minimal Sample Script

    Create sample_pytorch.py:

    import torch
import torch.nn as nn
import torch.optim as optim
from time import sleep

# Simple neural network
class Net(nn.Module):
    def __init__(self):
        super(Net, self).__init__()
        self.fc1 = nn.Linear(20, 10)
        self.fc2 = nn.Linear(10, 1)

    def forward(self, x):
        return self.fc2(torch.relu(self.fc1(x)))

# Setup
model = Net().to("cuda")
optimizer = optim.Adam(model.parameters())
criterion = nn.MSELoss()

# Training data
X = torch.randn(1000, 20, device="cuda")
y = torch.randn(1000, 1, device="cuda")

# Training loop
for epoch in range(5):
    optimizer.zero_grad()
    loss = criterion(model(X), y)
    loss.backward()
    optimizer.step()

    print(f"Epoch {epoch}, Loss: {loss.item():.4f}")
    sleep(2)  # Allow ACB to observe idle time

    Using Your Own Code

    ACB's Automatic Mode works with existing PyTorch code without modifications. Simply run your programs through agarun with AGA_ENABLE_AUTO=1.

    Submit the Job

    AGA_ENABLE_AUTO=1 agarun python sample_pytorch.py

    Expected Output: 

    Epoch 0, Loss: 1.0234
Epoch 1, Loss: 0.9876
Epoch 2, Loss: 0.9543
...

    Step 4: Test Multi-Job GPU Sharing

    To see ACB's dynamic GPU allocation in action, submit multiple jobs:

    AGA_ENABLE_AUTO=1 agarun python sample_pytorch.py >log1.txt &
AGA_ENABLE_AUTO=1 agarun python sample_pytorch.py >log2.txt &
AGA_ENABLE_AUTO=1 agarun python sample_pytorch.py >log3.txt &

    ACB will automatically schedule these jobs across available GPUs, maximizing utilization.

    What's Happening?

    • Jobs request GPU access during active computation
    • ACB allocates GPUs dynamically based on availability
    • GPUs are released during idle periods (e.g., sleep())
    • Multiple jobs share GPUs efficiently without conflicts

    Step 5: Stop the GPU Assigner

    When you're done, stop the service:

    gpu_assigner stop

    Expected Output: 

    INFO: Successfully killed GPU assigner process running on 127.0.0.1:11234

    Troubleshooting

    GPU Assigner Won't Start

    Problem: ERROR: GPU assigner is already running

    Solution: Stop the existing instance first: 

    gpu_assigner stop
gpu_assigner start

    License Errors

    Problem: License validation failed

    Solutions: - Verify ACB_KEY environment variable is set correctly - Ensure license.lic is in the current working directory - Check your license hasn't expired

    Connection Errors

    Problem: Error: Failed to communicate with the gpu assigner

    Solutions: - Verify GPU Assigner is running: gpu_assigner status - Check if port 11234 is available: netstat -an | grep 11234 - Restart GPU Assigner: gpu_assigner stop && gpu_assigner start

    Jobs Not Using GPU

    Problem: Jobs run but don't use GPU

    Solutions: - Ensure AGA_ENABLE_AUTO=1 is set - Verify CUDA is available: python -c "import torch; print(torch.cuda.is_available())" - Check GPU Assigner logs: cat $HOME/.acb/gpu-assigner.log

    Quick Reference

    Essential Commands

    # GPU Assigner
gpu_assigner start              # Start the service
gpu_assigner status             # Check status
gpu_assigner stop               # Stop the service

# Running Jobs
AGA_ENABLE_AUTO=1 agarun python script.py              # Single job
AGA_ENABLE_AUTO=1 agarun -- python script.py --args   # With arguments
AGA_REQUEST_DEVICE=cuda agarun python script.py       # Force GPU

# Logs
cat $HOME/.acb/gpu-assigner.log     # GPU Assigner logs
cat $HOME/.acb/job-history.log      # Job execution history

    Environment Variables

    Variable Purpose Example
    AGA_ENABLE_AUTO Enable Automatic Mode 1
    AGA_REQUEST_DEVICE Request specific device cuda, cpu, any
    AGA_GPU_ALLOC_SERVER_PORT Custom port 12345
    ACB_KEY Cloud license key your-api-key
